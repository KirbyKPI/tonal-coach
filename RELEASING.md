# Releasing

This repository uses Git tags and GitHub Releases for source releases.

## Release policy

- `main` is the integration branch
- Releases are tagged as `vX.Y.Z`
- Until `1.0.0`, minor releases may include breaking changes
- Every release should update [CHANGELOG.md](CHANGELOG.md)

## Maintainer checklist

1. Confirm CI is green on `main`
2. Review the draft release notes generated by Release Drafter
3. Update [CHANGELOG.md](CHANGELOG.md) from `Unreleased`
4. Bump the version in [package.json](package.json) if needed
5. Create and push an annotated tag such as `v0.2.0`
6. Publish the GitHub Release using the generated notes plus any manual edits
7. Move new changes back under `Unreleased`

## Suggested commands

```bash
npm version patch --no-git-tag-version
git add package.json package-lock.json CHANGELOG.md
git commit -m "chore: prepare vX.Y.Z"
git tag -a vX.Y.Z -m "vX.Y.Z"
git push origin main --follow-tags
```

## Notes

- Release Drafter prepares notes from merged pull requests, but maintainers are
  responsible for the final wording and categorization.
- If a release contains security fixes, coordinate disclosure timing with
  [SECURITY.md](SECURITY.md).
